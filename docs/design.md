# TORI Design Direction

TORI should look and feel like a sleek handheld technical instrument. It should be minimal, durable, and direct, with a strong emphasis on speed and readability.

## Industrial Design Language

Use these cues:

- Compact handheld proportions.
- Clean chamfered or beveled forms.
- Dark neutral shell materials.
- High-contrast display.
- Sparse markings and labels.
- Purposeful physical controls.
- Clear separation between camera, screen, scan button, and D-pad.

Avoid these cues:

- Combat-oriented names or labels.
- Decorative warning stripes unless they serve a real status purpose.
- Aggressive styling that distracts from the tool function.
- Busy sci-fi decoration.
- Oversized branding.

## Interface Language

The screen UI should be dense but calm:

- Use short labels.
- Prefer status codes and icons where space is tight.
- Keep primary data large enough to read quickly.
- Keep secondary data compact.
- Use high contrast for active state, selected item, and errors.
- Avoid long explanatory copy on-device.

## Core Screen States

Phase 1 should support these display states:

- Booting.
- Wi-Fi connecting.
- Server discovery.
- Ready to scan.
- Capturing.
- Uploading.
- OCR processing.
- Match found.
- Quantity editing.
- Updating quantity.
- Update success.
- Error.

## Controls

Recommended physical controls:

- Primary scan button: capture and submit an image.
- D-pad up/down: adjust quantity.
- D-pad left/right: move through candidate matches or change adjustment step in later phases.
- Confirm/select: submit quantity update.
- Back/cancel: return to ready state or dismiss errors.

## Visual Tone

Preferred descriptors:

- Technical.
- Minimal.
- Utilitarian.
- Rugged.
- Precise.
- Modern.
- Field-ready.

Avoid descriptors:

- Battle-themed product language.
- Aggressive gear marketing language.
- Claims that imply official field certification unless the device has been tested for it.

## Display Layout Defaults

The default Phase 1 match screen should reserve space for:

- Match confidence or status at the top.
- Item name as the primary text.
- Item ID as secondary text.
- Current quantity as a large editable value.
- Button hints along the bottom edge.

Text should be clipped, wrapped, or abbreviated deliberately. It should never overlap other UI regions.
